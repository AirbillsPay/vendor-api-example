---
title: "GET /network-checker — Detect Mobile Network by Phone"
sidebarTitle: "Network Checker"
description: "Detect the Nigerian mobile network for a phone number before airtime or data purchases. Returns MTN, Glo, 9mobile, or Airtel to set networkId in /transact."
---

Before sending airtime or data, use this endpoint to automatically detect the recipient's mobile network. Knowing the correct network upfront lets you populate the required `networkId` field in your `/transact` request and prevents failed transactions caused by mismatched carrier details.

## Endpoint

```
GET /api/vendor/gateway/network-checker
```

## Request

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `secretkey` | ✓ | Your business secret key |

### Query Parameters

<ParamField query="phone" type="string" required>
  The Nigerian mobile phone number to check, e.g. `"08012345678"`.
</ParamField>

## Response

<ResponseField name="status" type="string">
  `"00"` on success. Any other value indicates an error.
</ResponseField>

<ResponseField name="message" type="string">
  A human-readable result message, e.g. `"Successful"`.
</ResponseField>

<ResponseField name="data.network" type="string">
  The detected network name for the supplied phone number. Possible values: `"MTN"`, `"Glo"`, `"9mobile"`, `"Airtel"`.
</ResponseField>

<ResponseField name="data.network_id" type="string">
  The `networkId` value to pass directly in the `data` object of your `/transact` request. Corresponds 1:1 with `data.network` — see the [mapping table](#network-id-mapping) below.
</ResponseField>

<ResponseField name="data.error" type="string">
  Present only when the network could not be detected for the supplied phone number. In this case, `data.network` and `data.network_id` default to `"MTN"` and `"01"` — check for `data.error` before trusting the detected network.
</ResponseField>

### Example Response

```json
{ "status": "00", "message": "Successful", "data": { "network": "MTN", "network_id": "01" } }
```

## Code Example

```typescript
const params = new URLSearchParams({ phone: '08012345678' });
const response = await fetch(`${BASE_URL}/network-checker?${params}`, {
  headers: { 'secretkey': process.env.SECRET_KEY },
});
const { data } = await response.json();

if (data.error) {
  // Network could not be detected — data.network / data.network_id
  // have defaulted to "MTN" / "01". Don't blindly trust them.
  console.warn(`Could not detect network, defaulted to MTN: ${data.error}`);
}

console.log(data.network); // 'MTN'
console.log(data.network_id); // '01'
```

## Network ID Mapping

Pass `data.network_id` from the response directly in the `data` object of your `/transact` request — no manual mapping needed. For reference, it corresponds to:

| Network | `networkId` |
|---------|-------------|
| MTN | `"01"` |
| Glo | `"02"` |
| 9mobile | `"03"` |
| Airtel | `"04"` |

<Tip>
  Use `data.network_id` from the `/network-checker` response as-is. Skipping this step or guessing the network may cause the transaction to fail.
</Tip>
